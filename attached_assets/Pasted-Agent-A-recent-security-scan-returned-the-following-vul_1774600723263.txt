How it works in Youvian:

Database — There's a features table that stores each feature with a name, description, and status field. The status can be "requested", "confirmed", or "deployed".

Admin panel — You (as admin) can create new feature cards, and change their status using a dropdown. When you change a status:

"requested" → appears in the "Vote on Features" section (users can upvote/downvote)
"confirmed" → automatically moves to the "Coming Soon" section
"deployed" → automatically moves to the "Available Now" section
Voting system — There's also a feature_votes table so logged-in users can vote on requested features, and vote counts are tracked.

Build a full Feature Roadmap system with admin management:

Database:

Create a features table with columns: id (auto-increment primary key), name (text), description (text), status (varchar — one of "requested", "confirmed", "deployed"), upvotes (integer, default 0), downvotes (integer, default 0), created_at (timestamp).
Create a feature_votes table with: id, userId, featureId (foreign key to features, cascade on delete), voteType ("upvote" or "downvote"). Index on userId and featureId.
API routes:

GET /api/features — list all features, optionally filter by status
POST /api/features — create a new feature (admin only)
PATCH /api/features/:id — update feature name, description, or status (admin only)
DELETE /api/features/:id — delete a feature (admin only)
POST /api/features/:id/vote — logged-in users can upvote or downvote
GET /api/features/:id/user-vote — get the current user's vote on a feature

Three sections that features automatically flow between based on their status:
"Available Now" — shows features with status "deployed" in a responsive grid (1-3 columns)

"Vote on Features" — shows features with status "requested", expandable cards with upvote/downvote buttons for logged-in users
Cards have a glassmorphism style (semi-transparent backgrounds with backdrop blur), hover elevation, and dark mode support
Admin panel (feature management section):

Form to create new features with name and description fields
List of all features with a status dropdown (requested / confirmed / deployed)
Changing the status dropdown automatically moves the card to the correct section on the public features page
